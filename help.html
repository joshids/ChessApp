How to Play

Making Moves

Enter moves using standard chess notation in the move input field:

  • Simple moves: e4, Nf3, O-O (castling)
  • Captures: exd5 (pawn captures on d5)
  • Check: Qh5+
  • Checkmate: Qh5#
  • Piece notation: K (King), Q (Queen), R (Rook), B (Bishop), N (Knight), P (Pawn - usually omitted)

Game Status

The game status displays important information such as:

  • Current player's turn (White or Black)
  • Check warnings
  • Checkmate announcements
  • Stalemate notifications

Reset Game

Click the "Reset" button to start a new game from the initial position. This clears all moves, captured pieces, and highlights.

Animation Speed

Control how fast pieces move across the board using the animation speed slider:

  • 0% - 90%: Faster than normal (instant to quick)
  • 100%: Normal speed (default)
  • 110% - 200%: Slower than normal (slow motion)

The speed is adjustable in 10% increments. Your preference is saved automatically.

Replay Mode

Switch to "Replay" mode to replay a series of moves automatically:

  1. Click the "Replay" button to switch modes
  2. Enter moves in the textarea, one per line (e.g., e4, e5, Nf3)
  3. Click "Start Replay" to begin the automatic replay
  4. The board will update for each move with smooth animations
  5. Click "Stop Replay" to pause or return to "Play" mode

Note: Replay mode validates all moves before starting. Invalid moves will be highlighted with error messages.

Captured Pieces

When a piece is captured, it automatically appears in the appropriate captured pieces area:

  • Top area: Shows pieces captured by the second player (e.g., "Black's Lost Pieces")
  • Bottom area: Shows pieces captured by the first player (e.g., "White's Lost Pieces")

Captured pieces are displayed with:

  • Chess piece symbols (‚ôü, ‚ôú, ‚ôû, ‚ôù, ‚ôõ, ‚ôö)
  • Count badges for duplicate pieces
  • Styling that matches your selected skin

The labels change dynamically based on your selected piece skin colors.

Move Highlighting

After each move, the source and destination squares are automatically highlighted with a diagonal stripe pattern:

  • Source square: The square where the piece started (slightly more opaque pattern)
  • Destination square: The square where the piece moved to (slightly less opaque pattern)

The highlight pattern uses the opposite square color of your selected skin to ensure visibility:

  • Light squares use the dark square color for the pattern
  • Dark squares use the light square color for the pattern

Only the last move is highlighted. Previous highlights are automatically cleared when a new move is made or when the game is reset.

Board Copy and Load

Copy the current board position to share it with others or report issues. You can also load a board position from a copied text.

Copying Board Position

To copy the current board position:

  1. Navigate to the "Board Management" section in the controls panel
  2. Click the "Copy Board" button
  3. The board position is copied to your clipboard in multiple formats

The copied text includes:

  • Human-readable format: Lists all pieces with their positions (e.g., "White: K on e1, Q on d1, ... Black: K on e8, ...")
  • JSON format: Machine-readable format for programmatic use (e.g., {"a1": "R", "a2": "P", ...})

Use cases: Share interesting positions on social media, report bugs with specific board states, or save positions for later analysis.

Loading Board Position

To load a board position from copied text:

  1. Navigate to the "Board Management" section
  2. Click the "Load Board" button
  3. Paste the copied board position into the dialog
  4. Click "Load" to apply the position

The load function supports multiple input formats:

  • JSON format: {"a1": "R", "a2": "P", "e1": "K", ...}
  • Text format: White: K on e1, Q on d1, ... Black: K on e8, ...
  • Simple format: a1:R a2:P e1:K ...

Note: Loading a board position will reset the game state (move history and captured pieces). The board will be set to the exact position specified, preserving only the current turn and castling rights.

Piece Notation

When copying or loading, pieces are represented using standard notation:

White: K, Q, R, B, N, P
Black: k, q, r, b, n, p

Uppercase letters represent white pieces, lowercase letters represent black pieces.

Technical Details

Chess Engine

This app uses chess.js for move validation and game state management. All moves follow standard chess rules.

Data Persistence

Your preferences (selected skin and animation speed) are saved in your browser's localStorage and will persist across sessions.

Browser Compatibility

This app works best in modern browsers that support ES6+ features, CSS Grid, and CSS Custom Properties.
